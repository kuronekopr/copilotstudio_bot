# 02_PII_and_Data_Minimization_Decisions

## PD-01: 原本画像非保存 (No Original Image Storage)

| 項目 | 内容 |
|------|------|
| **決定日** | 2025-01-10 |
| **決定責任者** | Security Lead |
| **リスク軽減** | 90% |

**決定**：元画像（ユーザーがアップロードした状態）は保存しない。前処理後の画像のみ処理・保存。

**理由**：
- PII 漏洩リスク最小化
- GDPR データ最小化要件対応
- ストレージコスト削減（80-90%）

**実装**：Append-only イベントログで前処理完了を記録

## PD-02: PII 文字列非保存 (No PII String Storage)

| 項目 | 内容 |
|------|------|
| **決定日** | 2025-01-12 |
| **決定責任者** | Security Lead |
| **リスク軽減** | 85% |

**決定**：検出された PII テキスト（メール、電話等）は DB に保存しない。ハッシュのみ保存。

**理由**：
- PII テキスト漏洩リスク排除
- ハッシュから原文復元は困難
- 重複検出は可能（salted hash）

**実装**：`sha256(pii_value + session_salt)`

## PD-03: バウンディングボックス非保存 (No Bounding Box Storage)

| 項目 | 内容 |
|-----|------|
| **決定日** | 2025-01-15 |
| **決定責任者** | Security Lead |

**決定**：PII 位置情報（bounding box）は保存しない。

**理由**：
- バウンディングボックス + 画像サイズから PII テキストを推測可能
- 追加のセキュリティリスク

**実装**：検出結果は「PII 存在（Yes/No）」のみ記録
